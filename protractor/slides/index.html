Why testing is so important?

Testing is about gaining confidence that your code does what you think it should do

Whats the ideia around E2E testing?

  • How would the users see your application?
  • Does my backend is comunicating with my frontend?
  • Can I release this code?
  • It's NOT a replacement for Unit Testing!

The dark side of E2E testing

  • It needs a spefic environment running
  • It's 'hard' to write
  • It's difficult to debug
  • It's hard to keep the tests up to date

Install

  2. (sudo) npm install -g protractor
  3. (sudo) webdriver-manager update
  4. webdriver-manager start

Writing tests

Write your tests using Jasmine and WebdriverJS

describe('by model', function() {
	it('should find an element by text input model', function() {
	  var username = element(by.model('username'));
	  var name = element(by.binding('username'));

	  username.clear();
	  expect(name.getText()).toEqual('');

	  username.sendKeys('Jane Doe');
	  expect(name.getText()).toEqual('Jane Doe');
	});
});

Protractor global variables

  • browser: browser.get()
  • element and by: element(by.model('yourName'))
  • protractor: protractor.Key

Basic example

// Let's run it?
describe('angularjs homepage', function() {
  it('should greet the named user', function() {
    browser.get('http://www.angularjs.org');

    element(by.model('yourName')).sendKeys('Julie');

    var greeting = element(by.binding('yourName'));

    expect(greeting.getText()).toEqual('Hello Julie!');
  });
});

Searching elements on the page

element() vs element.all()

Single element

element( by.binding('appName') );

Collection of elements

// clicks the 3th element
element.all( by.css('[ng-click="openPage()"]') ).get(2).click();

by.binding

In your test

element( by.binding('myModel') );

In your application

<span ng-bind="myModel"></span>
<!-- or -->
<span>{{myModel}}</span>

by.model

In your test

element( by.model('myModel') );

In your application

<input ng-model="myModel" />

by.repeater

In your test

element( by.repeater('user in users').row(0).column('name') );

In your application

<ul ng-model="user in users">
  <li>
      <span>{{user.name}}</span>
  </li>
</ul>

by.css

In your test

element( by.css('[ng-click="sendMail()"]') );

In your application

<button ng-click="sendMail()">Send mail!</button>

Searching elements best practices

  • Take advantage of AngularJS attributes using by.model, by.repeater, etc
  • Avoid using potential CSS attributes, mainly IDs and Classes.

Page Objects

var AngularHomepage = function() {
  this.nameInput = element(by.model('yourName'));
  this.greeting = element(by.binding('yourName'));

  this.get = function() {
    browser.get('http://www.angularjs.org');
  };

  this.setName = function(name) {
    this.nameInput.sendKeys(name);
  };
};

Hands On!

Go further!

Separate your tests in various test suites

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',
  capabilities: { 'browserName': 'chrome' },

  suites: {
    homepage: 'tests/e2e/homepage/**/*Spec.js',
    search: ['tests/e2e/contact_search/**/*Spec.js']
  },

  jasmineNodeOpts: { showColors: true }
};
Running specific suite of tests 
protractor protractor.conf.js --suite homepage

Enable multiCapabilities

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',

  multiCapabilities: [
	{
	  'browserName' : 'chrome'
	}, 
	{
	  'browserName' : 'firefox'
	}
  ],

  specs: ['example-spec.js'],

    showColors: true
  }
};

Using onPrepare

    'browserName': 'chrome'
  },

  onPrepare: function() {
     browser.driver.manage().window().setSize(1600, 800);

  }
};

Export xml results of your Automated Suites

(sudo) npm install jasmine-reporters
//config.js
exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',

  capabilities: { 'browserName': 'chrome' },

  onPrepare: function() {
    require ('jasmine-reporters');
    jasmine.getEnv().addReporter(
      new jasmine.JUnitXmlReporter('./', true, true)
    );
  },

  jasmineNodeOpts: {
    showColors: true
  }
};

Using params

Your config.js

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',

  capabilities: { 'browserName': 'chrome' },

  params: {
    timestamp : Date.now()
  },

  onPrepare: function() {
     browser.driver.manage().window().setSize(1600, 800);
  },
 
  jasmineNodeOpts: { showColors: true }
};

Using params

Your test

describe('angular homepage', function() {
  
  it('should add my name', function() {
    element( by.model('yourName') ).sendKeys('Ramon Victor ' + browser.params.timestamp);
    expect( element(by.binding('yourName') ).getText() )
    	.toEqual('Ramon Victor ' + browser.params.timestamp);
  });   

});

Using jasmineNodeOpts

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',

  capabilities: { 'browserName': 'chrome' },

  jasmineNodeOpts: {
    showColors: true,
    defaultTimeoutInterval: 30000,
    isVerbose: true,
    includeStackTrace: true
  }
};
